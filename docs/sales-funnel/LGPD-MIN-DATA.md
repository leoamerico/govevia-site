# LGPD-MIN-DATA — Minimização de dados (Funil Self‑Service)

## Objetivo

Definir o mínimo de dados para prospecção e geração de diagnóstico/pacote, reduzindo risco LGPD.

## Princípios

- **Minimização:** coletar só o necessário para diagnóstico e proposta.
- **Finalidade explícita:** diagnóstico (Gap Report) e proposta/minuta (CLM Package).
- **Preferir contato institucional:** e-mail de domínio oficial quando possível.
- **Retenção em laboratório:** dados versionados no repo devem ser lab-safe e expurgáveis.

## Regras

- Evitar dados sensíveis.
- Evitar CPF/RG, dados pessoais desnecessários e qualquer informação não pública.
- Limitações devem constar no Gap Report (`limitations`).
